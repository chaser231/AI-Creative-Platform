# AI Creative Platform — Короткая продуктовая выжимка по канвасу

> Формат: краткая продуктовая записка на 1-2 страницы.
> Цель: быстро объяснить, что уже умеет канвас, где его ценность и куда логично двигаться дальше.

---

## 1. Что это за канвас

Канвас AI Creative Platform — это не просто область для ручного редактирования баннера, а центральный продуктовый модуль платформы. Он соединяет три вещи:

- визуальный редактор в духе Canva / Figma,
- AI-генерацию контента,
- multi-format архитектуру, где один мастер-макет адаптируется под несколько размеров.

В интерфейсе это выражено через два режима:

- **Wizard**: быстрый старт с шаблоном, структурированными полями и preview;
- **Studio**: полный редактор с канвасом, слоями, свойствами, AI, версиями и экспортом.

---

## 2. Какая ценность уже есть сейчас

### 2.1 Для пользователя

Пользователь уже может:

- собрать баннер с нуля в Studio;
- стартовать от шаблона через Wizard;
- работать со слоями, фреймами и авто-лейаутом;
- генерировать тексты и изображения через AI;
- редактировать изображение AI-инструментами;
- адаптировать один макет под несколько форматов;
- сохранять версии, экспортировать результат и переиспользовать шаблоны.

То есть канвас уже закрывает не только “демо-сценарий”, но и реальный production-like цикл создания рекламного креатива.

### 2.2 Для бизнеса

С точки зрения продукта канвас уже даёт три сильных преимущества:

1. **Скорость производства**
   За счёт шаблонов, Wizard и AI-генерации пользователь быстрее приходит к первому пригодному варианту.

2. **Масштабирование креатива**
   Через master/instance и resize-систему один креатив можно вести сразу в нескольких форматах без полного ручного пересбора.

3. **Снижение порога входа**
   Не-дизайнер может зайти через Wizard и AI, а дизайнер — дотянуть макет в Studio без выхода в другой инструмент.

## 3. Что особенно сильное в текущей реализации

Несколько частей продукта уже выглядят зрелыми:

### 3.1 Глубина canvas-логики

В редакторе уже есть:

- вложенные `frame`,
- `auto-layout`,
- `clipContent`,
- snap guides,
- deep-select внутри frame,
- transform / resize,
- drag-and-drop по иерархии,
- клавиатурное редактирование.

Это уже полноценная редакторная модель, а не “упрощённый баннерный конструктор”.

### 3.2 Multi-format архитектура

Самая сильная инженерная часть канваса — master/instance модель:

- контент хранится в master,
- layout живёт в resize-specific instances,
- smart resize умеет подставлять layout из шаблона и контент из текущего проекта.

Это продуктово важно, потому что именно здесь платформа уходит от “одного баннера” к системному производству креативов.

### 3.3 AI-слой уже многослойный

AI в редакторе уже существует не как одна кнопка, а как три разные поверхности:

- быстрый `AIPromptBar`,
- `ImageEditorModal` для доработки картинки,
- `AIChatPanel` для более сложного агентного сценария.

Особенно сильна текущая image-edit часть: remove background, inpaint, text-edit, outpaint, reference images и выбор моделей.

## 4. Где продукт ещё не догнал реализацию

Сейчас главный разрыв не в том, что “канваса мало”, а в том, что его продуктовая упаковка и docs отстают от реального состояния.

### 4.1 Экспорт

В документации обещаны `PNG/JPEG/PDF/SVG`, но подтверждённый UI сейчас даёт:

- PNG,
- batch ZIP из PNG,
- layer export,
- template JSON.

Здесь нужно либо честно синхронизировать документацию, либо довести экспорт до обещанного уровня.

### 4.2 Форматы

В коде уже есть resize-архитектура и даже preset registry, но в пользовательском сценарии формат пока создаётся в основном вручную через width/height.

Продуктово не хватает красивого preset-based UX:

- Instagram Post,
- Story,
- Facebook Cover,
- IAB,
- видео-форматы.

### 4.3 AI agent

Документация создаёт ожидание “AI, который полноценно управляет канвасом”. В коде агент уже умеет:

- строить план,
- искать шаблоны,
- применять шаблон,
- генерировать текст и картинку,
- выполнять ограниченный набор canvas actions.

Но до полного “chat with canvas” здесь ещё есть дистанция.

### 4.4 Bulk editing

PropertiesPanel очень сильна для одного объекта, но почти не работает как batch inspector для нескольких выделенных слоёв. Для реальной дизайнерской скорости это заметный следующий шаг.

## 5. Куда логично развивать канвас дальше

Если смотреть продуктово, следующий фокус выглядит так:

### Приоритет 1. Догнать UX до уже существующей архитектуры

- preset-панель форматов,
- понятный Smart Resize flow,
- более продуктовый слой вокруг multi-format режима,
- bulk edit для multi-selection.

### Приоритет 2. Синхронизировать docs и ожидания

- привести функциональную документацию к текущей правде кода,
- отдельно обозначить реализованное и roadmap,
- убрать обещания, которые пока не доведены в UI.

### Приоритет 3. Расширять AI там, где уже есть база

- не делать “ещё один AI-блок”,
- а усиливать существующие поверхности:
  - AI chat actions,
  - apply-to-selection,
  - template-aware generation,
  - more direct canvas edits.

---

## 6. Главный вывод

Канвас AI Creative Platform уже выглядит как реальное ядро продукта, а не только как supporting feature для AI-генерации.

Его ключевая сила в том, что он уже соединяет:

- ручное визуальное редактирование,
- AI-генерацию и AI-редактирование,
- шаблоны,
- multi-format production workflow.

Главная задача следующего этапа — не “построить канвас с нуля”, а сделать так, чтобы UX, docs и продуктовая коммуникация соответствовали той глубине, которая уже есть в реализации.
