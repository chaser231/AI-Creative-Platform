# Canvas Gap Backlog

> Набор задач, родившихся из расхождений между текущей реализацией канваса, продуктовой документацией и ожидаемым UX.
> Источник: `documentation/canvas_functionality_analysis.md`

---

## 1. Принципы приоритизации

- `P1` — влияет на понятность продукта, критические сценарии или заметный user expectation gap.
- `P2` — усиливает скорость работы и приводит UX в соответствие с архитектурой.
- `P3` — улучшение качества, ясности или консистентности, но без срочной блокировки.

## 3. Acceptance Criteria по задачам

## `CANVAS-001` Preset formats in ResizePanel

- В `ResizePanel` есть выбор готовых форматов.
- Форматы сгруппированы как минимум на `Social`, `Display`, `Video`.
- Можно добавить preset одним кликом без ручного ввода размеров.
- Сохраняется возможность добавить custom format вручную.

## `CANVAS-002` Export truth alignment

- Выбрано одно из двух:
  - docs обновлены под текущую реализацию,
  - или добавлены новые форматы экспорта.
- В пользовательском тексте редактора больше нет ложного ожидания по неподдерживаемым экспортам.

## `CANVAS-003` Toolbar model decision

- На уровне продукта принято решение, как должны работать create-tools.
- UI и help-тексты соответствуют этому решению.
- Нет ситуации, когда интерфейс обещает draw-mode, а фактически делает instant insert, не объясняя этого.

## `CANVAS-004` Richer AI canvas actions

- AI chat умеет не только добавлять text/image и грузить template, но и редактировать больше layer props.
- Есть guardrails: действия ограничены поддерживаемым списком, ошибки понятны пользователю.
- В истории сообщений видно, что именно AI собирается изменить.

## `CANVAS-005` Bulk editing

- При multi-selection панель показывает общий inspector.
- Пользователь может минимум:
  - выровнять,
  - изменить opacity,
  - поменять fill там, где применимо,
  - сместить / масштабировать набор объектов.

## `CANVAS-006` Lock visibility in LayersPanel

- Пользователь видит, что слой заблокирован, не открывая context menu.
- Можно быстро lock/unlock без ухода в правый клик.

## `CANVAS-007` Smart Resize product UX

- Пользователь понимает, что произойдёт при smart resize, до применения.
- Mapping объясняется простым продуктовым языком.
- После применения пользователь видит, какие слоты остались пустыми и что с этим делать.

## `CANVAS-008` AI docs refresh

- Описаны реальные AI surfaces: prompt bar, image editor, chat panel.
- Перечень моделей, caps и refs отражает текущий `ai-models.ts`.
- AI-обещания в docs не шире фактического action registry.

## `CANVAS-009` Template versioning

- У шаблона появляется история версий или хотя бы revision model.
- Есть способ увидеть, чем новая версия отличается от предыдущей.

## `CANVAS-010` Docs status system

- В ключевых документах используется единая маркировка:
  - реализовано,
  - частично,
  - planned / roadmap.
- Читатель больше не путает текущее состояние продукта и целевое видение.
